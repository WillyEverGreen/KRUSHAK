# Contributing

Thanks for your interest in contributing to Krushak.

## Quick start

1. Fork the repo and create a feature branch.
2. Run `flutter pub get`.
3. Make changes with clear, focused commits.
4. Verify formatting and run tests if applicable.

## Code style

- Follow Dart and Flutter style guides.
- Keep changes scoped and add comments only when needed.

## Tests

- Run `flutter test` when changing logic.
- If tests are not applicable, explain why in the PR.

## Issues and PRs

- Use clear titles and descriptions.
- Link related issues when possible.
- Include screenshots or recordings for UI changes.

## Security

If you discover a security issue, do not open a public issue. Contact the maintainer directly.
